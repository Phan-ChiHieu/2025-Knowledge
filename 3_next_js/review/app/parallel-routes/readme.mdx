# Parallel Routes

- Parallel Routes allow you to simultaneously or conditionally render multiple pages, with independent navigation, in the same layout.

  > Parallel Routes cho phép bạn đồng thời hoặc có điều kiện hiển thị nhiều trang, với điều hướng độc lập, trong cùng một bố cục.

- Parallel Routes can be used for advanced routing patterns like [Conditional Routes](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes#conditional-routes) and [Intercepted Routes](https://nextjs.org/docs/app/building-your-application/routing/intercepting-routes).

  > Parallel Routes có thể được sử dụng cho các mẫu định tuyến nâng cao như Conditional Routes và Intercepted Routes.

- Try using the tabs in one parallel route to navigate. Notice the URL changes but the unaffected parallel route is preserved.

  > Hãy thử sử dụng các tab trong một tuyến song song để điều hướng. Lưu ý URL thay đổi nhưng tuyến song song không bị ảnh hưởng vẫn được giữ nguyên.

- Try using the browser's backwards and forwards navigation. Notice the browser's URL history state and active UI state is correctly synced.

  > Hãy thử sử dụng điều hướng ngược và tiến của trình duyệt. Lưu ý trạng thái lịch sử URL của trình duyệt và trạng thái UI đang hoạt động được đồng bộ hóa chính xác.

- Try navigating to a tab in one parallel route and refreshing the browser. Notice you can choose what UI to show parallel routes that don't match the initial URL.

  > Hãy thử điều hướng đến một tab trong một tuyến song song và làm mới trình duyệt. Lưu ý bạn có thể chọn giao diện người dùng nào để hiển thị các tuyến song song không khớp với URL ban đầu.
